Window Packing in Pixinsight:

There are a series of widgets, from spinBox, labels, textInputs that are available. Most window widgets though have a series of these. These are packed together to define how they are constructed in the window.

This section will cover an overview to the packing methodology in Qt, and inherited within Pixinsight.

The example we will look at is the following widget:

This simple window contains the following widgets:

  1. Dialog Widget
  2. Label Widget
  3. ViewList Widget
  4. MessageBox Widget (hidden at present)
  5. HorizontalSizer (not obvious, but will so )
  6. VerticalSizer  (again not obvious, but will become so)
  7. SpinBox Widget
  8. Edit Widget
  9. PushButton Widget
If we notice we have the following in rows:
  • label + ViewList
  • label + spinBox
  • label + Edit

These are "packed" together using the HorizontalSizer widget

Once we have the rows, we can then pack the in a VerticalSizer widget

The VertcalSizer is then placed in the overall Dialog Widget

i.e. in code:


//Create the individual Widgets

   /////////////////////////////////////////

   this.helpLabel = new Label( this );
   this.helpLabel.frameStyle = FrameStyle_Box;
   this.helpLabel.text = "Label with Frame:<b>" + TITLE + " v" + VERSION + "</b> &mdash; Example of Qt widets " +
      "in Pixinsight.<b> In bold</b> <i> In Italics</i> ."

   /////////////////////////////////////////

   this.exampleViewListLabel = new Label( this );
   this.exampleViewListLabel.text = "Label with no frame:";


   /////////////////////////////////////////

   this.exampleViewList = new ViewList( this );

   /////////////////////////////////////////

   this.exampleSpinBoxLabel = new Label( this );
   this.exampleSpinBoxLabel.text = "Label for SpinBox";

  /////////////////////////////////////////

  this.exampleSpinBox = new SpinBox( this );

   /////////////////////////////////////////

   this.exampleEditLabel = new Label( this );
   this.exampleEditLabel.text = "Label for Edit";

   /////////////////////////////////////////

   this.exampleEdit = new Edit( this );
   this.exampleEdit.text = format( "Default Edit String" );

   ////////////////////////////////////////

   this.ok_Button = new PushButton( this );
   this.ok_Button.text = "OK";
   this.ok_Button.icon = this.scaledResource( ":/icons/ok.png" );

   this.cancel_Button = new PushButton( this );
   this.cancel_Button.text = "Cancel";
   this.cancel_Button.icon = this.scaledResource( ":/icons/cancel.png" );

//Build the HorizontalSizers

   this.viewListHorizontalSizer = new HorizontalSizer;
   this.viewListHorizontalSizer.add( this.exampleViewListLabel );
   this.viewListHorizontalSizer.add( this.exampleViewList, 100 );


   this.exampleSpinBoxHorizontalSizer = new HorizontalSizer;
   this.exampleSpinBoxHorizontalSizer.spacing = 4;
   this.exampleSpinBoxHorizontalSizer.add( this.exampleSpinBoxLabel );
   this.exampleSpinBoxHorizontalSizer.add( this.exampleSpinBox );


   this.bkgColor_Sizer = new HorizontalSizer;
   this.bkgColor_Sizer.spacing = 4;
   this.bkgColor_Sizer.add( this.exampleEditLabel );
   this.bkgColor_Sizer.add( this.exampleEdit );

   this.buttons_Sizer = new HorizontalSizer;
   this.buttons_Sizer.spacing = 6;
   this.buttons_Sizer.addStretch();
   this.buttons_Sizer.add( this.ok_Button );
   this.buttons_Sizer.add( this.cancel_Button );

//Pack the HorizontalSizers into a VerticalSizer


   this.sizer = new VerticalSizer;
   this.sizer.add( this.helpLabel );
   this.sizer.add( this.viewListHorizontalSizer );
   this.sizer.add( this.exampleSpinBoxHorizontalSizer );
   this.sizer.add( this.bkgColor_Sizer );
   this.sizer.add( this.buttons_Sizer );


Note in the above as well as packing widget instances into the sizer, we also added in spacers i.e this.buttons_Sizer.spacing = 6 . This allows for a fixed spacing to be added between individual widgets.

Qt also have the concept of spring-like , variable spacers with a sizer: these are stretchers, and allow for the widgets to be moved to fill available space.

The simplest way to show is to give an example: The following dialog has its maximum width specified by the fileDialog row: it has 3 horizontalSizer, each with the same number of widgets, but whose widgets are less in width that the fileDialog row. A stretcher is placed in different places in the horizontalSizer placking.

Selecting Image to Be Active

All of the open image windows can be accessed through the javascript object ImageWindow

Running the following code we can get all of the imageWindows:

            allWindows = ImageWindow.windows;

            console.writeln(allWindows);

 

which returns:


[object ImageWindow],[object ImageWindow]


i.e. two images, as there are two images in the window


Given an Image, to make the active window use:

ImageWindow.bringToFront()


Remember, each ImageWindow can have multiple views. To extract the mainView ImageWindow.mainView will return it. i.e.

view = ImageWindow.mainView

The id of a view, will then get the name of that view: i.e.

var count;

for(count=0; count < allWindows.length ; count++) {

let view = allWindows[count].mainView;

console.writeln("ID: " + view.id);

}


So to make active, if the id is the required one, use the bringToFront() functions




An example of this can be located in the file ChangeActiveWindow.js
